Patient User Stories – Smart Clinic Management System
Register
As a Patient,
I want to register,
So that I can book appointments.

Acceptance Criteria:

Patient enters name, email, phone, password
Account is created successfully
Login
As a Patient,
I want to login,
So that I can access my account.

Acceptance Criteria:

Valid credentials allow login
Dashboard opens
Search Doctor
As a Patient,
I want to search doctors by specialization,
So that I can find the right doctor.

Acceptance Criteria:

Patient enters specialization
System shows matching doctors
Book Appointment
As a Patient,
I want to book appointment,
So that I can consult doctor.

Acceptance Criteria:

Patient selects doctor and time
Appointment is saved
Confirmation shown
View Appointments
As a Patient,
I want to view my appointments,
So that I can track my bookings.

Acceptance Criteria:

Shows all booked appointments



Doctor User Stories – Smart Clinic Management System
Login
As a Doctor,
I want to login to the system,
So that I can view my appointments.

Acceptance Criteria:

Doctor enters email and password
System validates credentials
Doctor dashboard opens
View Appointments
As a Doctor,
I want to view my scheduled appointments,
So that I can manage my patients.

Acceptance Criteria:

Displays list of appointments
Includes patient name and time
Sorted by date
View Daily Schedule
As a Doctor,
I want to see appointments for a specific date,
So that I can prepare my schedule.

Acceptance Criteria:

Doctor selects date
System shows appointments for that date



Admin User Stories – Smart Clinic Management System
Add Doctor
As an Admin,
I want to add new doctors to the system,
So that patients can book appointments with them.

Acceptance Criteria:

Admin can enter doctor name, email, phone, and specialization
Doctor is saved successfully in database
Confirmation message is displayed
View Doctors
As an Admin,
I want to view all doctors,
So that I can manage doctor records.

Acceptance Criteria:

System displays list of all doctors
Includes doctor name, specialization, and contact details
Delete Doctor
As an Admin,
I want to remove a doctor,
So that outdated or inactive doctors are not shown.

Acceptance Criteria:

Admin can select doctor
Doctor is removed from database
Success message shown
